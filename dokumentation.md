# GitHub Skills – Kurzdokumentation

## 1. Worum geht es?

Die beiden GitHub-Übungen („Introduction to GitHub“ und „Communicate using Markdown“) vermitteln grundlegende Funktionen von GitHub. Der erste Kurs behandelt das Arbeiten mit Repositories, Branches, Commits und Pull Requests. Der zweite Kurs zeigt die grundlegende Verwendung der Markdown-Befehlen für Dokumentation und Formatierung von Texten in GitHub.

## 2. Wichtigste Take-Aways

### 1. Repositories
Ein Repository dient als zentraler Speicherort für Code und Dokumente. Grundlegende Aktionen sind das Anlegen, Bearbeiten und speichern in Versionen von Dateien.

### 2. Branches und Pull Requests
Branches ermöglichen es, Änderungen unabhängig vom Hauptzweig zu entwickeln (somit bleibt die "main"-Struktur sicher). Pull Requests dienen dem Überprüfen und Zusammenführen dieser Änderungen.

### 3. Commits
Ein Commit speichert eine Änderung inklusive einer kurzen, präzisen Nachricht. Dadurch bleiben Änderungen auch im Nachhinein nachvollziehbar.

### 4. Markdown-Grundlagen
Markdown bietet einfache Formatierungen wie:
- `#` für Überschriften  (unterschiedliche Header-Ebenen)
- `![Alt-Text](URL)` für Bilder  
- Codeblöcke mit drei Backticks  
- Checklisten (`- [ ]`)  

Markdown wird häufig für Dokumentationen, README-Dateien und Issues verwendet.

### 5. GitHub Actions
GitHub Actions ermöglichen automatisierte Workflows. In den Übungen werden sie genutzt, um Schritte automatisch zu prüfen, sobald Änderungen vorgenommen werden.

## 3. Fazit

Die beiden GitHub-Übungen vermitteln grundlegende Arbeitsweisen mit GitHub und Markdown. Sie zeigen, wie Änderungen strukturiert durchgeführt und dokumentiert werden können und wie Markdown für eine einfache Dokumentation eingesetzt werden kann.
